# AI를 활용한 업무 도구 개발 사례: Issue Manager

## 1. 배경 및 문제점

### 기존 업무 방식

업체에서 문제점 리스트를 받으면 **Comment를 하나하나 확인**해야 했습니다. Comment를 통해서만 지금 어떤 상태인지, 우리쪽에서 도움이 필요한 문제는 뭔지, 해결 예정인 건은 얼마나 되는지, 재현은 성공했는지 등을 파악할 수 있었습니다.

문제는 **업체가 Comment를 성실하게 적어놓지 않으면 확인 자체가 불가능**했다는 것입니다. 그때마다 업체 담당자에게 따로 물어보거나, 과거 메일을 뒤져보며 상태를 추적해야 했습니다.

이를 개선하기 위해 업체에 **문제별 Status를 추가해서 보내달라**고 요청했고, 그제서야 각 문제가 어떤 단계(분석중/재현중/코드리뷰 등)에 있는지 체계적으로 파악할 수 있게 되었습니다.

하지만 Status가 추가되면서 새로운 문제들이 생겼습니다:

- **수동 비교 작업**: 시스템에서 Excel을 다운로드받아서, 업체 문서에서 해결된 문제를 찾아 지우고, 새로 할당된 문제를 찾아 추가하는 작업을 수동으로 해야 했음
- **문제 추적 불가**: 하나하나 따로 대응하기에 급급하고, 전체적으로 몇 개가 해결됐고, Reopen은 몇 개고, 신규 할당은 몇 개인지 파악이 안 됨
- **이력 관리 불가**: 각 문제가 어떤 상태에서 며칠 머물렀는지 기록 자체가 없음
- **다중 과제 관리 혼선**: 여러 업체/과제를 동시에 관리할 때 파일이 뒤섞이는 실수 발생

### 소요 시간 (Before)
| 작업 | 소요 시간 | 비고 |
|------|----------|------|
| 일일 문제점 비교 및 전달 문서 작성 | **30분~1시간** | 시스템 Excel ↔ 업체 문서 수동 비교, 삭제/추가 |
| 문제점 이력 추적 | **불가** | 기록 자체가 없음 |
| 포스트모텀 분석 | **불가** | 각 상태별 소요 기간 데이터 없음 |

## 2. 해결 방법: AI와 대화로 도구 개발

### 접근 방식
- **코딩 경험 없이** Claude AI와 대화하며 웹 기반 문제점 관리 도구를 개발
- "이런 기능이 필요해" → AI가 설계/구현 → 실제 테스트 후 피드백 → 개선
- 약 **1주일** 동안 v0.3.0 → v1.9.2까지 **34번의 버전 업데이트**를 거쳐 발전

### 개발 과정 예시

**요청 1 - 설계 단계**: "여러 과제를 관리하려면 어떻게 해야할까? UX 전문가 그리고 시스템 아키텍트 입장에서 검토해보고 알려줄래?"
- → AI가 두 관점에서 분석: 단일DB vs 프로젝트별DB 비교표 제시
- → 장단점 논의 후 단일DB + project_id 방식 채택, 바로 구현

**요청 2 - UX 개선**: "Status Timeline이 한눈에 잘 안들어오는데... 좋은 방법이 없을까?"
- → 막대 차트 → 간트 차트 → 점+선 타임라인으로 **3번 진화**
- → 매번 실제 스크린샷 확인 후 "글자가 겹쳐 보인다" 등 구체적 피드백

**요청 3 - 실수 방지**: "실수로 다른 프로젝트에 파일을 올리는 경우가 있을것 같은데... 어떻게 예방하지?"
- → AI가 여러 방안 제시 (확인 팝업, 랜딩 화면, 2단계 Compare)
- → 논의 후 **프로젝트 선택 랜딩 + Preview → Save 2단계** 모두 채택

**요청 4 - 업무 로직**: "vendor 문서에 없고 system에 있는 문제가 Rejected면 Reopened로 처리해야 하는데 가능할까?"
- → AI가 compare 로직 수정 방안 제시, 같은 날 재비교 시 꼬이는 케이스까지 고려
- → `first_seen_date` 기반 판정 로직으로 모든 케이스 커버

### 핵심 포인트
1. AI에게 "구현해줘"만 하지 않고, **"어떤 방법이 좋을까?"**라고 먼저 물어봄
2. UX 전문가, 시스템 아키텍트 등 **역할을 부여**해서 다각도 검토
3. 결과물을 **직접 테스트**하고 문제점을 **구체적으로** 피드백
4. 실수 가능성, 엣지 케이스를 **미리 질문**해서 견고한 구조 설계

## 3. 결과물: Issue Manager

### 3.1 핵심 워크플로우

**Before (매일 30분~1시간):**
```
시스템에서 Excel 다운로드 → 업체 Excel과 눈으로 비교 → 
새 문제/해결된 문제 수동 파악 → 코멘트 수동 복사 → 
업체 전달용 Excel 수동 작성
```

**After (매일 2~3분):**
```
시스템에서 Excel 다운로드 → Issue Manager에 업로드 → Compare 클릭 →
결과 확인 → Save → Export to Excel → 업체에 전달
```

업체 파일은 코멘트 업데이트가 있을 때만 같이 올리면 됩니다. 기존 코멘트와 담당자 정보는 DB에서 자동 유지됩니다.

### 3.2 주요 기능

#### 문제점 자동 비교 (Compare)
- 시스템 Export 파일만 올리면 **자동으로 분류**: 신규 할당(Assigned), 해결(Resolved), 재발(Reopened)
- 업체 파일은 선택적 (코멘트 업데이트 시에만)
- 시스템에서 Rejected 상태인 문제는 자동으로 Reopened 처리
- **Preview → Save 2단계**로 잘못된 파일 업로드 시 피해 방지

#### 현황 대시보드
- **Summary 카드**: Total / Assigned / Reopened / Resolved 한눈에 확인
- **도넛 차트**: Status, Module, Owner별 분포 시각화
- **Issue Trend**: 날짜별 문제 현황 추이 (선 그래프)
- **Status Timeline**: 각 문제의 상태 변화를 점+선으로 시각화 (주말 음영 구분)
- **Stalled Issues**: 3일(Warning) / 7일(Critical) 2단계 지연 감지

#### 포스트모텀 지원
- 모든 문제의 **상태 변경 이력이 자동 기록**됨
- 각 문제가 **어떤 상태에서 며칠 머물렀는지** 추적 가능
- **Export Postmortem** 기능으로 전체 이력을 Excel로 추출
- 과제별로 상태가 달라도 OK (A과제: 3단계, B과제: 10단계 → 각자 기록대로 출력)

```
포스트모텀 Excel 예시:
| ID    | Headline | 분석중 | 재현중 | 코드리뷰 | Total | Resolved | Reopened |
|-------|----------|--------|--------|----------|-------|----------|----------|
| DF001 | 이슈A    |   3d   |   -    |    5d    |  8d   |    1     |    0     |
| DF002 | 이슈B    |   2d   |   4d   |    7d    | 13d   |    2     |    1     |
```

#### 프로젝트 관리
- **멀티 프로젝트**: 여러 과제를 독립적으로 관리 (데이터 격리)
- **업체별 그룹핑**: 업체별로 프로젝트를 묶어서 관리
- **랜딩 화면**: 프로젝트 선택 후 진입 (잘못된 프로젝트 작업 방지)
- **Milestones**: 과제 단계별 D-day 관리 (색상으로 긴급도 표시)
- **Project Notes**: 담당자, 연락처, 링크 등 자유 메모

#### 공유 및 배포
- **네트워크 공유**: exe 1대만 실행하면 팀원은 브라우저로 접속 (설치 불필요)
- **Copy for Confluence**: 클릭 한 번으로 보고서를 메일/Confluence에 붙여넣기
- **Export to Excel**: 현재 문제 목록을 코멘트 포함 Excel로 다운로드

#### 데이터 안전
- **자동 백업**: 하루 1회 DB 자동 백업 (30일 보관)
- **Preview 모드**: Compare 결과를 미리 확인 후 저장 (실수 시 취소 가능)
- **프로젝트 간 격리**: 파일 입력이 프로젝트 전환 시 자동 초기화

### 3.3 소요 시간 비교

| 작업 | Before | After | 절감 |
|------|--------|-------|------|
| 일일 문제점 비교 및 전달 문서 작성 | 30분~1시간 | **2~3분** | **90%+** |
| 현황 파악 (통계/분포) | 별도 집계 안 함 | **대시보드에서 바로 확인** | - |
| 메일/Confluence 공유 | 별도 작성 안 함 | **Copy 클릭 10초** | - |
| 문제점 이력 추적 | 불가 | **자동 기록** | - |
| 포스트모텀 분석 | 불가 | **Excel 한 번 클릭** | - |

## 4. 향후 계획

### 시스템 API 연동 (자동화)
현재는 시스템에서 수동으로 Excel을 다운로드하여 업로드하는 방식이지만, **시스템 API와 연동**하여 이 과정도 자동화할 계획입니다.

```
현재:  시스템 → (수동 다운로드) → Excel → Issue Manager → Compare
향후:  시스템 → (API 자동 연동) → Issue Manager → 자동 Compare + 알림
```

이렇게 되면 사용자가 별도로 파일을 올릴 필요 없이, **자동으로 최신 상태가 갱신**되고, 새로운 문제 할당이나 Stalled 이슈 발생 시 **알림**도 가능해집니다.

## 7. AI 활용 팁 (배운 점)

### 효과적인 AI 활용법
1. **"구현해줘"보다 "어떻게 하면 좋을까?" 먼저** 
   - AI에게 전문가 역할(UX, 아키텍트)을 부여하면 더 나은 설계를 얻을 수 있음
   
2. **작게 시작해서 점진적으로 개선**
   - 처음부터 완벽하게 만들려 하지 말고, 핵심 기능부터 → 테스트 → 피드백 → 개선
   
3. **직접 테스트하고 구체적으로 피드백**
   - "안 돼요" ✕ → "이 버튼 누르면 이렇게 나오는데 이건 왜 그런 거야?" ○
   - 스크린샷을 첨부하면 더 정확한 수정 가능
   
4. **엣지 케이스와 실수 가능성을 미리 질문**
   - "같은 날에 여러 번 하면 어떻게 돼?", "다른 프로젝트 파일을 잘못 올리면?"
   - AI가 예방 구조까지 함께 설계

### 이런 분들에게 추천
- 반복적인 Excel 비교/집계 작업이 있는 분
- 여러 업체/과제의 이슈를 동시에 관리하는 분
- 보고서 작성에 시간을 많이 쓰는 분
- 코딩 경험이 없지만 업무 자동화를 하고 싶은 분
