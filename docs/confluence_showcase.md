# AI를 활용한 업무 도구 개발 사례: Issue Manager

## 1. 배경 및 문제점

### 기존 업무 방식
- 업체에서 받은 문제점 Excel과 시스템에서 Export한 Excel을 **수동으로 비교**
- 어떤 문제가 새로 들어왔는지, 해결됐는지, 다시 열렸는지 **눈으로 하나하나 확인**
- 여러 과제를 동시에 관리할 때 **파일이 뒤섞이는 실수** 발생
- 각 문제가 어떤 상태에서 며칠 머물렀는지 **이력 추적 불가**
- 주간 보고 시 **수작업으로 통계 집계** (Status별 개수, Owner별 분포 등)

### 소요 시간 (Before)
| 작업 | 소요 시간 |
|------|----------|
| 일일 문제점 비교/업데이트 | 30~60분 |
| 주간 보고용 통계 집계 | 1~2시간 |
| Confluence/메일 보고서 작성 | 30분 |
| 문제점 이력 추적 | 수동 관리 불가 |

---

## 2. 해결 방법: AI와 대화로 도구 개발

### 접근 방식
- **코딩 경험 없이** Claude AI와 대화하며 웹 기반 도구를 개발
- "이런 기능이 필요해" → AI가 설계/구현 → 테스트 후 피드백 → 개선
- 총 **2일** 만에 v0.3.0 → v1.9.2까지 발전

### 개발 과정 예시

**요청 1**: "여러 과제를 관리하려면 어떻게 해야할까? UX 전문가 그리고 시스템 아키텍트 입장에서 검토해보고 알려줄래?"
- → AI가 두 관점에서 분석 후 DB 설계안 제시
- → 승인 후 바로 구현

**요청 2**: "Status Timeline이 한눈에 잘 안들어오는데... 좋은 방법이 없을까?"
- → 막대 차트 → 간트 차트 → 점+선 타임라인으로 3번 진화
- → 매번 스크린샷 확인 후 피드백

**요청 3**: "실수로 다른 프로젝트에 파일을 올리는 경우가 있을것 같은데... 어떻게 예방하지?"
- → AI가 여러 방안 제시 (확인 팝업, 랜딩 화면, 2단계 Compare)
- → 논의 후 **Preview → Save 2단계 구조** 채택

### 핵심 포인트
- AI에게 "구현해줘"만 하지 않고, **"어떤 방법이 좋을까?"**라고 먼저 물어봄
- UX 전문가, 시스템 아키텍트 등 **역할을 부여**해서 다각도 검토
- 결과물을 **직접 테스트**하고 문제점을 구체적으로 피드백

## 3. 결과물: Issue Manager

### 주요 기능

#### 3.1 문제점 자동 비교 (Compare)
- 시스템 Export 파일만 올리면 자동으로 새로운 문제, 해결된 문제, 재발된 문제 분류
- 업체 파일은 선택적 (코멘트 업데이트 시에만)
- **Preview → Save 2단계**로 실수 방지

#### 3.2 대시보드
- **Summary 카드**: Total / Assigned / Reopened / Resolved 한눈에 확인
- **도넛 차트**: Status, Module, Owner 분포 (라벨 직접 표시)
- **Issue Details**: 전체 문제 목록 + Excel Export
- **Status Timeline**: 각 문제의 상태 변화를 점+선으로 시각화 (주말 음영 표시)
- **Issue Trend**: 날짜별 추이 선 그래프
- **Stalled Issues**: 3일(Warning) / 7일(Critical) 2단계 감지

#### 3.3 프로젝트 관리
- **멀티 프로젝트**: 과제별 독립 관리
- **그룹핑**: 업체별로 프로젝트 묶기
- **랜딩 화면**: 프로젝트 선택 후 진입 (잘못된 프로젝트 방지)
- **Milestones**: 단계별 D-day 관리 (색상으로 긴급도 표시)
- **Project Notes**: 담당자/연락처 메모 (URL 자동 링크)

#### 3.4 공유 및 배포
- **네트워크 공유**: exe 1대만 실행, 팀원은 브라우저로 접속
- **Copy for Confluence**: 클릭 한 번으로 보고서 복사
- **Export to Excel**: 현재 문제 목록 다운로드
- **Export Postmortem**: 전체 이력 + 상태별 소요일 Excel

#### 3.5 데이터 안전
- **자동 백업**: 하루 1회 DB 자동 백업 (30일 보관)
- **New vs Reopened 자동 판정**: 시스템 Rejected 상태 자동 감지
- **프로젝트 간 격리**: 다른 프로젝트 데이터와 절대 섞이지 않음

### 소요 시간 (After)
| 작업 | Before | After |
|------|--------|-------|
| 일일 문제점 비교/업데이트 | 30~60분 | **2~3분** (파일 올리고 Compare) |
| 주간 보고용 통계 집계 | 1~2시간 | **0분** (대시보드에서 바로 확인) |
| Confluence/메일 보고서 작성 | 30분 | **10초** (Copy for Confluence 클릭) |
| 문제점 이력 추적 | 불가 | **자동** (Timeline + Postmortem) |

## 6. 배운 점

### AI 활용 팁
1. **"구현해줘"보다 "어떻게 하면 좋을까?" 먼저** - AI에게 전문가 역할을 부여하면 더 나은 설계를 얻을 수 있음
2. **작게 시작해서 점진적으로** - 처음부터 완벽하게 만들려 하지 말고, 핵심 기능부터 구현 후 피드백하며 개선
3. **직접 테스트하고 구체적으로 피드백** - "안 돼요"보다 "이 버튼 누르면 이렇게 나오는데 이건 왜 그런 거야?"
4. **실수 방지 구조를 요청** - "이런 실수를 할 수 있을 것 같은데 어떻게 방지하지?"

### 이런 분들에게 추천
- 반복적인 Excel 비교/집계 작업이 있는 분
- 여러 업체/과제의 이슈를 동시에 관리하는 분
- 보고서 작성에 시간을 많이 쓰는 분
- 코딩 경험이 없지만 업무 자동화를 하고 싶은 분
